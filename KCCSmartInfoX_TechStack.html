Backend Frameworks & Libraries
⚡
FastAPI
Latest stable
High-performance Python web framework. Handles all REST API routes: /api/chat, /api/admin/*, /api/knowledge, /api/concerns, and more.
Core web framework — REST API server
🦄
Uvicorn
ASGI Server
Lightning-fast ASGI server that runs the FastAPI application. Supports hot-reload in development. Deployed with --host 0.0.0.0 for network access.
Application server — runs FastAPI app
🗃
SQLAlchemy
ORM + Core
Python SQL toolkit and ORM. Defines database models (ChatLog, Announcement, Subscriber, AdminUser, Concern) and handles all database operations.
Database ORM — models & queries
🔐
python-jose
JWT / cryptography
JSON Web Token (JWT) library used for admin authentication. Issues signed tokens on login, verifies them on every protected route.
Authentication — JWT token generation & verification
🔑
passlib (bcrypt)
bcrypt 4.0.1
Password hashing library. Hashes admin passwords using bcrypt before storing. Used to securely verify passwords during login without storing plaintext.
Security — password hashing & verification
📄
python-multipart
File upload support
Enables FastAPI to accept multipart/form-data file uploads. Required for PDF, DOCX, TXT document uploads and announcement image uploads.
File uploads — documents & images
⚙️
python-dotenv
Environment config
Loads environment variables from the .env file (GROQ_API_KEY, EMAIL_USER, EMAIL_PASSWORD, SECRET_KEY) so sensitive credentials are never hardcoded.
Configuration — secrets & environment vars
🚦
slowapi
Rate limiting
Rate limiting middleware for FastAPI. Prevents API abuse by limiting how many requests a client can make per minute to chat and admin endpoints.
Security — request rate limiting
📧
smtplib (stdlib)
Python built-in
Python's built-in email library. Used with Gmail SMTP to send announcement emails to subscribers. Runs in background tasks to avoid blocking the API.
Email — announcement delivery to subscribers
📑
pdfplumber
PDF extraction
Extracts plain text from PDF files page by page. Used during document ingestion to read uploaded PDFs before chunking and embedding.
Document processing — PDF text extraction
📝
python-docx
DOCX extraction
Reads Microsoft Word (.docx) files and extracts paragraph text. Used during document ingestion alongside pdfplumber for multi-format support.
Document processing — Word file text extraction
Artificial Intelligence & Machine Learning
🤖
Groq API
llama-3.1-8b-instant
Cloud LLM inference API. Generates AI answers from retrieved context (RAG). Also rewrites student queries to clean English. Ultra-fast inference via Groq hardware.
LLM — answer generation, query rewriting, greetings
🧠
sentence-transformers
all-MiniLM-L6-v2
Local embedding model that converts text into 384-dimensional vectors. Runs entirely on-device. Used to embed both documents (ingestion) and student questions (search).
Embeddings — semantic similarity search
🗂
ChromaDB
PersistentClient
Open-source vector database. Stores document chunk embeddings and performs cosine similarity search to find the top 3 most relevant chunks for each question.
Vector store — knowledge base storage & retrieval
🔁
RAG Pipeline (Retrieval-Augmented Generation)
Custom-built in pipeline.py
The core AI logic — not a third-party library but a custom pipeline built on top of the above tools. Flow: spell fix → LLM query rewrite → embed question → ChromaDB search → distance threshold check → trim context → Groq LLM generation → bullet format cleanup → is_answered check → log to database.
Core AI — ties together all ML tools into the chat answer flow
Frontend Frameworks & Libraries
⚛️
React
Version 18
Component-based UI library. All pages (Landing, Chat, Admin Dashboard, Analytics, etc.) are React components using hooks (useState, useEffect, useRef).
UI framework — all pages and components
⚡
Vite
Version 5
Next-generation frontend build tool. Provides instant dev server with HMR (Hot Module Replacement) and optimized production builds. Proxy config routes /api/* to backend.
Build tool — dev server & production bundler
🎨
TailwindCSS
Version 3
Utility-first CSS framework. All styling done with classes (no custom CSS files). Custom KCC colors (kcc-blue, kcc-gold, kcc-dark) and animations (fadeIn, float, slideDown) defined in tailwind.config.js.
Styling — all UI visual design
🗺
React Router DOM
v6
Client-side routing. Manages navigation between pages: / (Landing), /chat, /admin/login, /admin/dashboard, /admin/knowledge, /admin/analytics, and more.
Routing — page navigation & URL management
📡
Axios
HTTP client
Promise-based HTTP client. Makes all API calls from the frontend to the FastAPI backend using relative paths (/api/...). Handles file uploads and JWT token headers.
API calls — frontend-to-backend communication
📊
Recharts
Chart library
React charting library. Powers the Analytics page: daily question bar chart, feedback pie chart, and answer rate trend. Built on SVG for clean rendering.
Data visualization — analytics charts
✨
Lucide React
Icon library
Clean, consistent SVG icon set. Used throughout the system for UI icons: MessageSquare, AlertCircle, ThumbsUp/Down, Download, Printer, QrCode, and more.
Icons — all UI iconography
🔔
React Hot Toast
Notification toasts
Lightweight toast notification library. Shows success, error, and info messages after admin actions (uploading documents, posting announcements, saving settings).
UX — success/error notification feedback
📝
React Markdown
Markdown renderer
Renders AI-generated markdown in the chat. Formats bullet lists, bold text, and paragraphs so AI answers display cleanly instead of raw markdown characters.
Chat — AI answer formatting & rendering
📱
react-qr-code
QR generator
Generates SVG QR codes from a URL. Used in the Admin QR Code page to create scannable codes pointing to the chatbot URL for students to scan on campus.
QR Code — campus access via phone scan
Database & Storage
🗄
SQLite
kccsmartinfox.db
Lightweight embedded relational database. Stores: ChatLogs, Announcements, Subscribers, AdminUsers, Concerns, SchoolInfo, OfficePolicies. No separate DB server needed.
Primary database — all structured data
🔍
ChromaDB
Local vector store
Vector database stored at backend/knowledge_base/vectorstore/. Persists document embeddings to disk. Loaded on backend startup as a singleton to prevent SQLite lock conflicts.
AI database — document embeddings & semantic search
📁
Local Filesystem
backend/knowledge_base/
Uploaded PDF/DOCX/TXT files saved to backend/knowledge_base/documents/. Announcement images saved to backend/knowledge_base/announcement_images/. Served as static files.
File storage — uploaded documents & images
Browser APIs (Built-in — No Library Needed)
🎙
Web Speech API
SpeechRecognition
Built-in browser API for voice-to-text. The mic button in the chat uses window.SpeechRecognition to transcribe student speech directly in the browser (Chrome required).
Voice input — speech-to-text in chat
📋
Clipboard API
navigator.clipboard
Built-in browser API. Used in the QR Code admin page to copy the chatbot URL to clipboard with the Copy button, showing a checkmark confirmation.
UX — copy URL to clipboard
🖼
Canvas API
HTMLCanvasElement
Used in QR Code download feature. Converts the SVG QR code to a PNG image with white margin (quiet zone) via canvas.toDataURL() for download.
QR export — SVG to PNG conversion for download
