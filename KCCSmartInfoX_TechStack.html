KCCSmartInfoX — Technology Stack

Kabankalan Catholic College | AI-Powered Campus Information System | Capstone Project 2025–2026

Backend Server-side · Python · FastAPI
🚀

FastAPI

v0.110+

Main web framework that powers all REST API endpoints — chat, admin, announcements, knowledge base, school info, and office processes. Chosen for its automatic API documentation (Swagger UI at /docs), async support, and fast performance.

REST APIAsyncSwagger UI
🌐

Uvicorn

ASGI Server

The ASGI (Asynchronous Server Gateway Interface) server that runs the FastAPI application. Started with uvicorn main:app --reload. Enables high-concurrency request handling needed for simultaneous chat sessions.

ASGIProduction Server
🗄️

SQLite + SQLAlchemy

SQLAlchemy ORM

SQLite is the relational database that stores all structured data: chat logs, announcements, subscribers, knowledge documents, admin users, audit logs, concerns, and school info. SQLAlchemy is the ORM used to define models and query the database without writing raw SQL.

DatabaseORM10 Tables
🧠

ChromaDB

Vector Database

Vector store used to index all knowledge base documents for semantic search. When a student asks a question, the query is embedded and compared against ChromaDB to retrieve the most relevant document chunks. A singleton client prevents SQLite lock conflicts.

Vector StoreRAGSemantic Search
🔢

Sentence Transformers

all-MiniLM-L6-v2

Pre-trained NLP model used to convert text (both documents and student questions) into numerical embeddings (vectors). These vectors are stored in ChromaDB and used to find semantically similar content. Runs locally — no API cost for embeddings.

EmbeddingsNLPLocal Model
⚡

Groq API

llama-3.1-8b-instant

Large Language Model (LLM) service used to generate natural language answers from retrieved document context (RAG pipeline). Also used for query rewriting (understanding Taglish/typos) and the admin AI assistant. Chosen for its free tier and fast inference speed.

LLMRAGQuery Rewrite
📄

pdfplumber

PDF Parser

Used in the knowledge base ingestion pipeline to extract raw text from uploaded PDF files (e.g., KCC College Handbook). Once extracted, the text is chunked and embedded into ChromaDB so the AI can answer questions about the document's content.

PDF ParsingKnowledge Base
📝

python-docx

DOCX Parser

Extracts text from Microsoft Word (.docx) files uploaded to the knowledge base. Works alongside pdfplumber to ensure the system accepts multiple document formats (PDF, DOCX, TXT) for the RAG pipeline.

DOCX ParsingKnowledge Base
🔐

Passlib + bcrypt

bcrypt 4.0.1

Used to securely hash admin passwords before storing them in the database. When an admin logs in, the entered password is verified against the stored hash. bcrypt is an industry-standard hashing algorithm resistant to brute-force attacks.

SecurityPassword Hashing
🎫

python-jose

JWT Authentication

Generates and verifies JSON Web Tokens (JWT) for admin authentication. After a successful login, a JWT is issued to the admin browser. All protected admin routes (knowledge base, announcements, chat logs, etc.) require a valid token in the request header.

JWTAdmin AuthSecurity
📧

smtplib (Gmail SMTP)

Python Standard Library

Sends email notifications to subscribers when a new announcement is posted. Uses Gmail's SMTP server with an app password stored in .env. Email sending is offloaded to FastAPI's BackgroundTasks to avoid blocking the API response.

EmailNotificationsBackground Task
🔤

pyspellchecker

Spell Checker

Performs a fast word-level spell correction on student queries before they are sent to the LLM for rewriting. Custom word lists are loaded for KCC-specific terms (BSIT, BSBA, TOR, enrollment, etc.) and Filipino words so they are never incorrectly "corrected."

NLPQuery Processing
Frontend Client-side · JavaScript · React
⚛️

React 18

v18

The core JavaScript UI library used to build the entire frontend — chat interface, admin panel, landing page, and all user/admin pages. React's component model allows reusable UI pieces (cards, modals, sidebar) across all pages.

UI FrameworkSPAComponents
⚡

Vite 5

v5

Build tool and development server for the React app. Chosen over Create React App for its significantly faster hot-module replacement (HMR) — changes appear in the browser instantly without a full page reload. Also handles production bundling.

Build ToolDev ServerHMR
🎨

TailwindCSS 3

v3

Utility-first CSS framework used for all styling — dark/light mode, responsive layout, animations, and the KCC color theme (navy blue + gold). Allows rapid UI development directly in JSX without writing separate CSS files.

CSS FrameworkDark ModeResponsive
🔗

Axios

HTTP Client

Used by all frontend pages to send HTTP requests to the FastAPI backend (chat, login, knowledge base CRUD, announcements, analytics, etc.). Handles request headers (JWT token injection) and JSON response parsing automatically.

HTTP ClientAPI CallsAuth Header
🗺️

React Router DOM

v6

Handles client-side navigation between pages without full page reloads. Used to define all routes: landing page, chat page, admin login, and all admin panel sub-pages (dashboard, knowledge base, announcements, analytics, subscribers, chat logs, etc.).

RoutingNavigationSPA
📊

Recharts

Chart Library

Used in the Admin Analytics page to render visual charts — bar charts for top questions, line charts for chat activity over time, and pie charts for answer rates. Built on top of SVG and React, making it easy to embed interactive charts in the dashboard.

AnalyticsChartsAdmin Dashboard
🔷

Lucide React

Icon Library

Provides clean, consistent SVG icons used throughout the system — nav icons in the admin sidebar, action buttons (edit, delete, upload, copy), chat UI icons (send, mic, globe), and status indicators. Replaces the need for separate icon font files.

IconsUI Components
🔔

React Hot Toast

Notification Library

Displays brief pop-up notifications (toasts) for user feedback — success messages when knowledge is added, error alerts on failed uploads, and confirmation messages when announcements are posted or deleted. Appears as small non-blocking banners.

NotificationsUX Feedback
📝

React Markdown

Markdown Renderer

Renders the AI's responses as formatted text in the chat window. The Groq LLM returns answers using markdown syntax (bold text with **, bullet lists with - , numbered steps with 1. 2. 3.). React Markdown converts this into proper HTML so the chat shows nicely formatted lists, headings, and bold terms — not raw symbols. A custom bullet-cleaning function also runs on the backend to ensure consistent formatting before the answer reaches the frontend.

MarkdownChat UIAI ResponsesFormatting
