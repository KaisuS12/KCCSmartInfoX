AI Improvements
Thank-You Instant Reply AI
The AI now detects gratitude messages before any other processing. When a student says "thank you," "thanks," "salamat," "maraming salamat," or similar expressions, the system responds immediately with a warm, randomized reply — no document search needed.
  • Supports both English and Filipino/Bisaya variants
  • Detected before the greeting check for natural feel
  • 5 different reply templates to avoid repetition
Groq API Fallback / Error Handling AI
All Groq LLM calls are now wrapped in try/except blocks. If the Groq API is temporarily unavailable or returns an error, the student sees a friendly message asking them to try again — instead of a raw server error. Greeting responses also have their own fallback.
FAQ Auto-Generator AI
The Analytics page can generate suggested knowledge base entries from unanswered student questions using the Groq AI. The admin clicks Generate, reviews the AI-suggested content, and can add it to the knowledge base in one click — directly closing the knowledge gap.
Office Processes Auto-Sync to AI AI
Every time an office process is added, edited, or deleted in the admin panel, the change is automatically re-synced to the ChromaDB vector store. The AI can immediately answer student questions about office procedures without requiring a manual document re-upload.
Export Unanswered Questions CSV — Fixed Fix
The Export button in Chat Logs and Analytics now correctly downloads a CSV file. Previously, the browser's window.open() could not attach an Authorization header, causing a "Not authenticated" error. Fixed by accepting the JWT token as a query parameter instead.
New Admin Features
Admin Login Audit Log Admin
Every admin session event is now recorded automatically:
  • Login Success — successful login
  • Login Failed — wrong password attempt
  • Logged Out — admin clicked Logout
  • Session Timeout — auto-logged out after inactivity
Each entry shows the device type (Windows PC, iPhone, Android, etc.) and IP address. Visible in the Settings page under Admin Audit Log. Last 50 events are stored.
Backend Online / Offline Indicator Admin
The Dashboard header now shows a real-time status pill indicating whether the FastAPI backend is reachable. A green pulsing dot means Online; a red dot means Offline. Useful for local WiFi demo setups where multiple devices share one backend server.
Clickable Filter Cards — Concerns UX
The three summary cards on the Student Concerns page (All Concerns, Pending, Resolved) are now clickable filter buttons. Clicking a card instantly filters the list below. The selected card highlights in color. The separate "All" filter button was removed for a cleaner layout.
Clickable Filter Cards — Chat Logs UX
Same UX improvement for the Chat Logs page. The Total Questions, Answered, and Unanswered summary cards at the top now act as filter buttons. Clicking Answered filters the list to only answered logs, clicking Unanswered shows only unanswered ones, and clicking Total Questions resets to all. Replaces the old filter pill row.
Edit Announcements Admin
Admin can now edit existing announcements via a pencil icon. A modal prefilled with the current title, body, image, and schedule dates opens. Changes are saved with a PUT request and the list refreshes immediately.
Inline Edit Knowledge Base Entries Admin
Text entries in the Knowledge Base can be edited directly on the page. When saved, the old indexed content is removed from ChromaDB and replaced with the new content automatically — no re-upload required.
Security Upgrades
Auto-Logout on Inactivity (30 min) Security
The admin panel now automatically logs out the administrator after 30 minutes of inactivity. Activity is tracked via mouse movement, keystrokes, clicks, scroll, and touch events. Each activity event resets the timer. When the timer fires, a "timeout" event is logged in the audit log and the admin is redirected to the login page.
Rate Limiting on Admin Login Security
The admin login endpoint is now rate-limited to 5 requests per minute per IP address using the slowapi library. This prevents brute-force password attacks. Exceeding the limit returns HTTP 429 Too Many Requests.
CORS Restricted to Localhost Origins Security
The backend CORS policy was tightened from allow_origins=["*"] (any origin) to only:
  • http://localhost:5173
  • http://localhost:4173
  • http://127.0.0.1:5173
  • http://127.0.0.1:4173
This prevents unauthorized cross-origin access to the API.
File Upload Size Limit (20 MB) Security
Single and bulk knowledge base file uploads now enforce a 20 MB per-file size limit. Files larger than 20 MB are rejected with an HTTP 413 error before being saved, preventing large files from filling up the server's storage.
Rate Limiting on Chat Endpoint Security
The public /api/chat endpoint is rate-limited to 20 requests per minute per IP address. This prevents automated bots from flooding the system with queries and consuming Groq API credits.
UX Improvements & Bug Fixes
Character Counter on Chat Input User
The chat input now shows a live character counter below the text field. It displays the current count out of the 500-character maximum. The counter turns yellow at 350 characters and red at 450 to warn the student before they reach the limit.
Retry Button on Chat Error User
If the AI returns an error (network issue or Groq timeout), the error message now includes a Retry button. Clicking it automatically resends the same question without the student having to retype it.
404 Not Found Page UX
A custom 404 page is now shown for any unrecognized URL instead of a blank page or redirect. It displays the KCC logo, a "Page not found" message, and two navigation buttons: Go to Home and Open Chatbot.
Skeleton Loaders & Animations UX
Loading skeleton placeholders are shown on Dashboard, Concerns, and other pages while data is being fetched — replacing the empty blank flash. Smooth animations added: KCC logo floats in chat, announcements panel slides in, concern modal fades in, stat cards lift on hover.
QR Code Port Auto-Detection Fix
The QR Code Generator now reads the current browser port from window.location.port instead of hardcoding 4173. This means the generated QR URL always matches the actual running server — port 5173 in development or 4173 in preview — so students who scan the code can always reach the chat.
PWA — Installable on Mobile User
KCCSmartInfoX is now configured as a Progressive Web App (PWA). Students can install the chatbot directly to their phone's home screen from the browser. Works on both Android and iOS (Safari). App icon and splash screen use the KCC logo.
